# File Upload Server

Express + TypeScript file upload service. Docker Compose binds the container upload
directory `/data/uploads` to `SAVE_FILE_PATH` on the Docker host.

## API

- `GET /health`: health check.
- `POST /upload`: upload one or more files with `multipart/form-data`.
- `GET /files`: list saved files.
- `GET /files/:fileName`: download a saved file.

Example:

```bash
curl -F "file=@./demo.png" http://localhost:3000/upload
```

## Local Development

```bash
npm install
npm run dev
```

## Local Docker

```bash
cp .env.example .env
mkdir -p uploads
docker compose up --build -d
```

Files are saved to `./uploads` by default. Change `SAVE_FILE_PATH` in `.env` to
persist files somewhere else on the Docker host.

## GitHub Actions Deployment

The workflow `.github/workflows/deploy.yml` does:

1. Install dependencies and build TypeScript.
2. Build and push a Docker image.
3. Copy `docker-compose.prod.yml` and `.env` to the remote server.
4. Run `docker compose pull` and `docker compose up -d` remotely.

Configure these GitHub Actions variables:

- `IMAGE_REGISTRY`: image registry host, for example `ghcr.io`.
- `IMAGE_NAMESPACE`: image namespace, for example `owner`.
- `SAVE_FILE_PATH`: absolute host path for persisted uploads, for example `/data/file-upload-server/uploads`.
- `APP_PORT`: optional host port, default `3000`.
- `DEPLOY_PATH`: optional remote compose directory, default `/opt/file-upload-server`.
- `UPLOAD_MAX_FILE_SIZE`: optional max bytes per file, default `104857600`.
- `UPLOAD_MAX_FILES`: optional max files per request, default `10`.
- `UPLOAD_ALLOWED_MIME_TYPES`: optional comma-separated allow list, for example `image/png,image/jpeg`.
- `PUBLIC_BASE_URL`: optional public base URL used in returned file URLs.

Configure these GitHub Actions secrets:

- `IMAGE_REGISTRY_USERNAME`
- `IMAGE_REGISTRY_PASSWORD`
- `SERVER_HOST`
- `SERVER_USER`
- `SERVER_PASSWORD`
- `SERVER_PORT`: optional, default `22`.

The remote server must have Docker and the Docker Compose plugin installed. The
workflow logs in to the image registry on the server before pulling the new image.

Image names are generated by the workflow:

```text
${IMAGE_REGISTRY}/${IMAGE_NAMESPACE}/${package.json name}:${current branch suffix}
```

For example, branch `feature/upload-api` builds
`ghcr.io/owner/file-upload-server:upload-api`.
